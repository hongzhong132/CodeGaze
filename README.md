# CodeGaze

> 一个融合编程训练、代码提交、社区交流、AI 助手与考试监测的智能编程互助平台

## 一、项目概述

CodeGaze 是一个面向编程学习、训练与考核场景设计的智能编程互助平台。  
项目以编程题练习为核心，围绕“题目浏览—题目作答—代码提交—结果记录—交流讨论—智能辅助—过程监测”构建完整闭环，兼顾日常学习训练与正式考试作答两种使用场景。

本项目基于 **Python + Django** 构建后端服务，结合 **Vue 3** 前端技术、**MySQL** 数据存储以及 **DeepSeek 大模型能力**，实现了题目训练平台与智能辅助功能的深度融合，具有较强的实践意义与应用价值。

---

## 二、项目背景

随着程序设计课程、算法训练平台和在线考试系统的普及，传统编程平台在以下方面仍存在不足：

- 题目训练与学习辅助相互割裂
- 平台通常只关注“提交结果”，缺乏对“作答过程”的支持
- 学生在独立训练时，缺少即时、智能化的思路引导与代码分析
- 编程考试场景中，过程约束与辅助监测能力较弱
- 交流讨论、练习记录与智能反馈之间缺乏统一闭环

为解决上述问题，本项目尝试构建一个同时支持**学习训练**与**考试管理**的智能编程平台，通过引入大模型辅助分析与 AI 监测机制，提升平台的智能化水平与应用适配能力。

---

## 三、项目目标

本项目旨在设计并实现一个集以下能力于一体的智能编程互助平台：

1. 提供题目浏览、搜索、详情查看与收藏管理功能
2. 支持多语言代码提交与提交记录展示
3. 支持练习模式与考试模式双场景切换
4. 集成 AI 助手能力，实现题意解释、代码分析、问题定位与优化建议
5. 支持考试场景中的 AI 监测展示与过程辅助约束
6. 提供社区讨论与交流功能，增强学习互动性
7. 为后续扩展判题系统、排行榜、成长体系和云端部署打下基础

## 五、项目特色

### 1. 双模式作答设计
平台区分**练习模式**与**考试模式**：

- 练习模式强调学习、调试、验证与 AI 辅助
- 考试模式强调规范作答、过程约束与监测辅助

这种设计使平台同时适用于日常训练和正式考核。

### 2. AI 助手深度融入编程学习流程
平台集成大模型能力，可围绕当前题目与代码场景提供：

- 题目思路解释
- 代码分析
- bug 定位
- 优化建议

从而提升用户的学习效率与代码理解能力。

### 3. AI 监测增强考试场景适配性
在考试模式下，平台支持 AI 监测状态展示，可结合摄像头、人脸状态等信息进行辅助监测，提升作答过程的规范性。

### 4. 学练评交流一体化
平台不仅支持题目训练和提交记录，还提供社区讨论与互动交流功能，形成较为完整的学习闭环。

### 5. 面向扩展的系统结构
项目采用较清晰的模块化设计，后续可继续扩展判题、推荐、排行榜、用户成长体系与服务器部署能力。

## 六、创新点

### 1. 将编程训练与考试监测相结合
相较于传统平台仅关注提交结果，本项目进一步关注作答过程，增强了考试场景下的平台适配能力。

### 2. 将大模型能力引入编程互助平台
通过引入 DeepSeek 系列大模型，平台具备代码解释、代码分析、问题诊断与优化建议等智能能力。

### 3. 构建练习 / 考试双场景切换机制
通过统一平台承载两类不同作答场景，提高了系统的复用性与实用性。

### 4. 兼顾学习辅助与过程约束
平台既强调 AI 助手对学习效率的提升，也重视考试模式下过程规范管理，体现了“智能辅助”与“行为约束”的平衡。

## 七、系统功能模块

### 1. 用户模块
负责用户登录、身份识别、个人信息展示等基础功能。

### 2. 题目模块
负责题目列表展示、搜索、标签分类、题目详情展示、题目收藏等功能。

### 3. 提交模块
负责多语言代码提交、提交记录保存、模式区分与页面展示。

### 4. AI 助手模块
负责接入大模型能力，为用户提供题意解释、代码分析、问题定位和优化建议。

### 5. AI 监测模块
负责考试场景下的监测状态展示、人脸检测信息反馈与过程辅助监测。

### 6. 社区交流模块
负责用户讨论、题目交流与内容互动，增强平台学习氛围。

### 7. 后台管理模块
负责题目维护、平台管理与系统运营支持。

## 八、核心页面展示说明

### 1. 题目列表页
题目列表页用于集中展示平台中的题目资源，支持关键词搜索、收藏筛选与卡片式浏览。  
用户可快速查看题目名称、难度、描述摘要和收藏状态，并进入题目详情页。

### 2. 题目详情页
题目详情页用于展示题目描述、输入输出格式、讨论区及统计信息。  
同时支持用户根据实际需求进入练习模式或考试模式，为后续代码提交提供入口。

### 3. 提交代码页
提交代码页支持多编程语言选择与代码输入。  
在练习模式下，页面可结合 AI 助手进行代码分析与学习辅助；在考试模式下，页面将更强调独立作答与行为规范。

### 4. AI 监测页 / 监测区域
考试模式下，平台右侧会展示 AI 监测状态区域，包括摄像头状态、人脸数量、检测后端、接口耗时等信息，用于辅助监测与过程展示。

## 十、系统架构说明

本项目整体采用前后端协同开发思路，核心架构如下：

- **前端层**：负责页面展示、交互逻辑与用户操作反馈
- **后端层**：负责业务逻辑、用户管理、题目管理、提交管理与接口服务
- **数据层**：使用 MySQL 存储用户、题目、提交记录、社区交流等核心数据
- **AI 服务层**：通过大模型接口为平台提供智能辅助能力
- **监测服务层**：为考试模式提供监测状态管理与检测信息反馈

##十三、本地部署说明
1. 克隆项目
git clone https://github.com/hongzhong132/CodeGaze.git
cd CodeGaze
2. 创建并激活虚拟环境
Windows
python -m venv .venv
.venv\Scripts\activate
macOS / Linux
python3 -m venv .venv
source .venv/bin/activate
3. 安装依赖
pip install -r requirements.txt
4. 配置环境变量

请在项目根目录创建 .env 文件，示例如下：

DJANGO_SECRET_KEY=your_secret_key
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost

CODEGAZE_DB_ENGINE=django.db.backends.mysql
CODEGAZE_DB_NAME=codegaze_db
CODEGAZE_DB_USER=your_db_user
CODEGAZE_DB_PASSWORD=your_db_password
CODEGAZE_DB_HOST=127.0.0.1
CODEGAZE_DB_PORT=3306

AI_ASSISTANT_PROVIDER=nvidia
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://integrate.api.nvidia.com/v1
AI_ASSISTANT_MODEL=deepseek-ai/deepseek-r1
AI_ASSISTANT_TIMEOUT=60
5. 创建数据库

在 MySQL 中执行：

CREATE DATABASE codegaze_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
6. 执行迁移
python manage.py makemigrations
python manage.py migrate
7. 创建管理员账号（可选）
python manage.py createsuperuser
8. 启动项目
python manage.py runserver

启动后访问：

http://127.0.0.1:8000/
